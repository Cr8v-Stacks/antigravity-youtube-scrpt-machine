# Selection Matrices

Use these matrices before writing a full production pack. They prevent random style choices and help each output choose the right system for the topic, opening, shot list, thumbnail, and production constraints.

## System Hierarchy

Choose in this order:

1. Channel/niche visual identity lock: the house style for this channel or topic family.
2. Episode visual system: the global visual world used across the video.
3. Identity preset: the setting-specific version of that visual system.
4. Shot visual type: the function of each individual visual beat inside that world.
5. Prompt variation pattern: how A/B/C prompt variants differ without breaking consistency.
6. Character consistency level: how stable people or archetypes need to be.
7. Adjacent content systems: hook type, narrative engine, research depth, thumbnail strategy, and pacing density.

Do not invent a new episode visual system when a shot visual type or prompt variation would solve the problem. Keep the global system stable; create variety through shot choices.
Do not output only a broad system name such as "Hybrid Sketch System Map" or "Cinematic Documentary System." Pair it with an identity preset so the look is consistent.

## Channel Visual Identity Lock

Every full production pack must choose a channel/niche identity lock before selecting shot prompts. The lock is the production style contract for the episode and thumbnail.

Return the lock as:

- Channel/niche identity.
- Default episode visual system.
- Identity preset name.
- Approved alternate or insert systems.
- Style DNA: texture, line style or lighting, palette, character treatment, object treatment, label treatment.
- Thumbnail art DNA: composition, text behavior, recurring symbols, contrast style.
- Forbidden drift: visual choices that would make the channel feel inconsistent.

Use the default visual system for most shots. Use approved alternates only as inserts, proof moments, or special segments unless the episode's whole story demands the alternate.

Important distinction:

- "Renewable / Clean Energy / Energy-History Domain Preset" is a domain implementation of Cinematic Documentary System, not a separate canon system.
- Evidence Board / Polaroid can be the whole episode system only for investigative stories. Otherwise, use it as a shot visual type inside Cinematic Documentary.
- Text cards, date cards, maps, quote cards, and diagrams are shot visual types, not episode visual systems.

## Prompt Variation Matrix

For long-form, each shot usually has A/B/C prompt variations. Variations should change execution, not the meaning.

| Variation type | What changes | What stays fixed | Use when |
|---|---|---|---|
| Framing variation | Close-up vs medium vs overhead | Same subject, action, visual system, narration beat | The shot has one strong object or person |
| Evidence variation | Object, document, map, or diagram version of the same proof | Same claim and timestamp | The beat can be proven visually in multiple ways |
| Emotion variation | Anxious, tense, reflective, skeptical expression | Same scene and props | Human reaction is the retention driver |
| Metaphor variation | Literal scene vs symbolic diagram vs comparison | Same concept and style | The narration is abstract but needs visual options |
| Complexity variation | Minimal card vs detailed scene vs hybrid diagram | Same beat and key text | Editors may need simpler or richer visuals |

Never vary by changing the selected visual system unless the shot is intentionally an insert and still matches the episode's style rules.

Premium variation pattern:

- Do not assign the same A/B/C skeleton to every shot.
- For each 5-10 shot cluster, deliberately rotate variation roles so the editor sees a mix of character action, object clue, map/timeline, social pressure, archive/document, split comparison, system loop, and final synthesis.
- Each prompt should have its own first visual decision after the selected style phrase; repeated openings such as "medium shot of..." across dozens of prompts are a warning sign.
- Consistency should come from the identity preset, palette, recurring characters, object anchors, and negative constraints, not from repeating one prompt grammar.

| Video angle | Thumbnail strategy | Visual ingredients |
|---|---|---|
| Myth-busting | Crossed-out wrong answer | Wrong object, red X, correct clue, 2-4 word text |
| Hidden system | Diagram loop | Human/object, arrows, trap/reward label |
| Forgotten invention | Evidence object | Machine/photo/document, date, mystery label |
| Scale shock | Number-first | Big number, tiny human/object comparison |
| Social behavior | Face/reaction | Awkward group moment, one strong emotion word |
| Business/tech pressure | Rival or risk | Product/factory/boardroom plus warning or cost cue |
| Ancient/history origin | Object-first | Tablet/coin/map/tool, date or "not what you think" label |

## Pacing Density Matrix

| Format | Script pacing | Visual pacing | Notes |
|---|---|---|---|
| Short under 60s | New beat every 3-8s | 8-18 visuals | No slow setup |
| Long-form 6-7 min | New pressure every 15-25s | 70-95 visuals | Use only when topic cannot sustain 8 min |
| Standard 8:00-9:30 | New pressure every 20-30s | 90-130 visuals | Default monetization-ready band; 8 minutes is the floor, not a rigid ceiling |
| 9-10+ earned expansion | New pressure every 20-30s | 115-150 visuals | Use when research and outline have real talking points, not filler |
| 10-12 min deep dive | New pressure every 20-35s | 140-190 visuals | Needs more evidence, chronology, conflict, or proof, not padding |

## Production Complexity Matrix

| Complexity | Use when | Output expectation |
|---|---|---|
| Lean | Fast production, simple concept, limited visual variety needed | Fewer systems, more text cards and diagrams, still timed tightly |
| Standard | Normal long-form production pack | Full script, 90-120 shot beats, 3 prompt variants, metadata |
| Premium | High-value documentary topic | More research anchors, character detail, evidence shots, stronger thumbnail A/B testing |
| Split delivery | Output would become too long | Deliver script and first shot-list part, then continue shot list in later parts |
